public composite HDFSWriteFormat(input inStream)

Convert a tuple to a string and then write to HDFS. The operator converts input tuples on its input stream to strings, and then sends them to the HDFS2FileSink. It includes a default conversion using SPL's tuple format, but allows you to supply a operator to do the conversion from tuple to rstring.

Parameters

  • formatter: (optional) operator to be used for converting from tuple to string
  • writer: (optional) operator to be used to write to HDFS
  • file: File name to be passed on to HDFSFileSink
  • hdfsUser:
  • timeFormat: passed to the file sink; used when the filename contains %TIME

Input Ports

  • inStream: Stream to be written to HDFS

public composite HDFSReadFormat(output myStream)

Read a line from HDFS and convert it to a tuple. The operator reads a line from HDFS and converts it to a tuple. By default, it use's SPL's cast function to convert from a tuple to a string.

Parameters

  • file: The file on hdfs to be read.
  • streamType:
  • initDelay: (optional) Specifies the time to wait before reading the file.
  • hdfsUser: (optional) The users to be used when connecting to HDFS.
  • decoder: (optional) The operator to use to convert a string to a tuple. By default, it uses the cast function.
  • reader: (optional) The operator to use to read HDFS lines.

Output Ports

  • myStream: the output stream of tuples

public composite HDFSReadFormatInputStream(output myStream; input filenames)

Read from HDFS with filenames supplied on the command line and covert strings to tuples. Treat each tuple on the input stream as a file name, open the file, and convert each line to a tuple.

Parameters

  • decoder: optional parameter to overrride default operator to use for converting from HDFS. It must accept a type parameter.
  • reader: optional parameter to override default operator to use for reading from HDFS.
  • initDelay: optional initial delay before reading files.
  • streamType: type of the output stream.
  • hdfsUser: optional parameter specifying user to connect to HDFS

Input Ports

  • filenames: a stream of rstrings representing files in HDFS

Output Ports

  • myStream: a stream of tuples. Each tuple corresponds to a line read from HDFS.
